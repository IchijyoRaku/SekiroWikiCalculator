## 3. 产品目标与边界

## 3.1 目标

在 MediaWiki 页面中实现可交互的“敌人数值计算器”，计算并展示：

1. HP 上限
2. 躯干上限
3. 躯干恢复速度
4. （可选）伤害倍率链路展示

支持维度：

1. 模式：主线 / 再战 / 死斗
2. 难度：普通 / 钟鬼 / 交符 / 双难
3. 天色：清晨 / 正午 / 黄昏 / 深夜 / 深夜+钟鬼（按敌人可用集合）
4. 周目：1~8（再战/死斗不使用周目）
5. 阶段：1阶段 / 2阶段 / 3阶段（按敌人 `buff_active_phase_num` 控制）
6. 敌人：按敌人列表切换

## 3.2 边界（本期不做）

1. 不做“基础数据展板”
2. 不做“伤害计算器”
3. 不做编辑器后台
4. 不在 Wiki 内部存放大数据库（采用外部托管拉取）

## 4. 核心业务对象与关系

## 4.1 主实体

1. `Enemy`（敌人聚合对象）
2. `NpcParamRow`（基础属性与特效入口）
3. `SpEffectRow`（倍率详情）
4. `BuffTable`（各类 buff ID -> 倍率）

## 4.2 关系总览

1. `Enemy.npc_param_id` 是全链路查询入口。
2. `npc_param` 给出基础值（`base_hp/base_stamina/staminaRecoverBaseVel`）及多个 buff 入口 ID。
3. 入口 ID 需要在对应 buff 表定位倍率；部分 buff 还需继续到 `speffect_param` 取 `hp_bonus`、`stamina_bonus`。
4. `NG_buff` 按周目和是否交符（序列偏移）决定成长倍率。
5. `enemy_type_buff` 仅在交符/双难生效。
6. `inner_battle_buff` 提供再战/死斗场地与模式乘区倍率。

## 4.3 表与职责对应

1. `npc_param`：敌人基础值 + 入口 ID
2. `speffect_param`：特效倍率明细（至少 `hp_bonus/stamina_bonus`）
3. `base_bonus_buff`：区域天色倍率（常规）
4. `base_bonus_buff_for_first_NG_no`：一周目交符专用区域倍率
5. `multi_NG_balance_buff`：多周目区域平衡倍率
6. `NG_buff`：周目成长倍率
7. `enemy_type_buff`：兵种倍率
8. `inner_battle_buff`：再战/死斗相关场地与模式倍率
9. `calculator`：Excel 对照样例（用于验算）

## 5. 数据映射与归一化规则

## 5.1 前端最终需要的标准字段（建议）

每个敌人归一化后至少包含：

1. `id` / `name`
2. `base_hp`
3. `base_stamina`
4. `base_stamina_recover`
5. `time_seq[]`
6. `base_bonus_buff_ids[]`
7. `base_bonus_buff_for_first_ng_no_charm_ids[]`
8. `multi_ng_balance_buff_ids[]`
9. `ng_buff_ids_or_index_rule`
10. `enemy_type_buff_id`
11. `special_buff_id`
12. `phase_change_buff_id`
13. `mind_battlefield_buff_ids[]`
14. `re_battle_buff_ids[]`
15. `death_battle_buff_ids[]`
16. `stamina_recover_down_at_hp_reduce_buff_ids[]`
17. `buff_active_phase_num`

## 5.2 关键索引规则

1. 天色序列建议统一索引：`0=清晨,1=正午,2=黄昏,3=深夜,4=深夜+钟鬼`
2. 难度索引采用“内部计算索引”与“页面展示顺序”分离：
`calc_index: 0=普通,1=双难,2=交符,3=钟鬼`（与再战/死斗 buff 数组顺序对齐）；
`display_order: 普通,钟鬼,交符,双难`（与示例表阅读顺序对齐）
3. 周目输入范围：`1~8`
4. 主线多周目计算时，周目倍率索引：
`normal: NG_buff[ng_num-1]`，`charm: NG_buff[ng_num-1+7]`

## 5.3 ID 关系（文档已给定）

1. 一周目区域天色 `-200 = 多周目平衡`
2. 区域天色 `+500 = 多周目平衡`
3. 一周目区域 `= 多周目平衡 +200 = 区域天色+700`
4. 可优先以 `multi_NG_balance_buff` 作为锚点反推其他表 ID

## 5.4 数据对齐结论（已按注释修订）

1. `npc_param.base_bonus_buff_for_first_NG_no_charm` 保持作为 `npc_param` 基础字段使用，不按 `enemy_type_buff` 解释。
2. 本期计算范围聚焦三项主指标：`hp`、`stamina`、`stamina_recover`。
3. `再战buff.md` 中“怨鬼死斗”区间更正为 `7982-7985`，后续实现以修正值为准。

## 5.5 字段命名映射（文档名 -> 数据库名）

| 文档字段 | 数据库表 | 数据库列名 | 说明 |
|---|---|---|---|
| `base_stamina_recover` | `npc_param` | `base_stamina_recover` | 躯干恢复基础值 |
| `base_hp` | `npc_param` | `base_hp` | HP 基础值 |
| `base_stamina` | `npc_param` | `base_stamina` | 躯干基础值 |
| `base_bonus_buff_id` | `npc_param` | `base_bonus_buff` | 区域天色倍率入口 |
| `base_bonus_buff_for_first_ng_no_charm_id` | `npc_param` + `base_bonus_buff_for_first_NG_no` | `multi_NG_balance_buff + 200`（推导） | 当前数据源中按 ID 关系推导一周目交符路径入口 |
| `multi_ng_balance_buff_id` | `npc_param` | `multi_NG_balance_buff` | 多周目平衡倍率入口 |
| `enemy_type_buff_id` | `npc_param` | `base_bonus_buff_for_first_NG_no_charm` | 本数据源中与 `enemy_type_buff` 对应使用 |
| `special_buff_id` / `phase_change_buff_id` / `mind_battlefield_buff_ids` / `re_battle_buff_ids` / `death_battle_buff_ids` | `npc_param` + `inner_battle_buff` + `speffect_param` | 当前 `整合表.sqlite` 无直接列 | 通过规则推导与外部映射补齐 |

## 6. 计算逻辑定义（用于前端实现）

## 6.1 统一变量

1. `B_hp/B_sta/B_rec`：基础 HP / 基础躯干 / 基础躯干恢复
2. `A_time`：区域天色倍率（来自 `base_bonus_buff`）
3. `A_time_1ng_charm`：一周目交符区域倍率（来自 `base_bonus_buff_for_first_NG_no`）
4. `A_multi`：多周目平衡倍率（来自 `multi_NG_balance_buff`）
5. `A_ng`：周目倍率（来自 `NG_buff`）
6. `A_enemy_type`：兵种倍率（来自 `enemy_type_buff`）
7. `A_mind/A_re/A_death`：场地 / 再战 / 死斗倍率（来自 `inner_battle_buff`）
8. `A_phase`：转阶段倍率（来自 `phase_change_buff` -> `speffect_param`）
9. `A_special_hp/A_special_sta`：特殊常驻倍率（来自 `special_buff` -> `speffect_param`）

## 6.2 主线（一周目）

1. 普通：`Base * A_time`
2. 钟鬼：`Base * A_time(time_seq+1)`
3. 交符：`Base * A_time_1ng_charm * A_enemy_type`
4. 双难：`Base * A_time_1ng_charm(time_seq+1) * A_enemy_type`

备注：交符/双难才应用 `A_enemy_type`。

## 6.3 主线（2~8周目）

1. 普通：`Base * A_time * A_multi * A_ng(normal_index)`
2. 钟鬼：`Base * A_time(time_seq+1) * A_multi(time_seq+1) * A_ng(normal_index)`
3. 交符：`Base * A_time * A_multi * A_ng(charm_index) * A_enemy_type`
4. 双难：`Base * A_time(time_seq+1) * A_multi(time_seq+1) * A_ng(charm_index) * A_enemy_type`

## 6.4 再战

1. 普通：`Base * A_mind[0] * A_re[0]`
2. 钟鬼：`Base * A_mind[3] * A_re[3]`
3. 交符：`Base * A_mind[2] * A_re[2]`
4. 双难：`Base * A_mind[1] * A_re[1]`

## 6.5 死斗

1. 普通：`Base * A_mind[0] * A_death[0]`
2. 钟鬼：`Base * A_mind[3] * A_death[3]`
3. 交符：`Base * A_mind[2] * A_death[2]`
4. 双难：`Base * A_mind[1] * A_death[1]`

## 6.6 阶段与特殊特效叠乘

1. 阶段判定采用 `buff_active_phase_num`（取值 `1/2/3`）：
当 `selected_phase >= buff_active_phase_num` 且敌人存在 `phase_change_buff` 时，额外乘 `A_phase`。
2. `special_buff` 对 HP 与躯干可能不同倍率，`hp_bonus` 与 `stamina_bonus` 必须分开乘。
3. `stamina_recover_down_at_hp_reduce_buff` 本期先不纳入动态血线分段计算，仅预留扩展位。

## 6.7 取整规则

1. 与 Excel 样例保持一致，最终显示值按 `INT` 向下取整。

## 7. 页面交互与展示需求

## 7.1 必要输入控件

1. 敌人下拉选择
2. 天色选择（按敌人可用天色动态启用，仅影响主线矩阵）
3. 阶段选择（`1/2/3`，按 `buff_active_phase_num` 动态启用）
4. 明细展开开关（控制乘区拆解是否展开）

## 7.2 输出区

1. 主线结果矩阵：`8 周目 x 4 难度`，每格展示 HP / 躯干 / 躯干恢复
2. 再战结果矩阵：4 难度并排展示
3. 死斗结果矩阵：4 难度并排展示
4. 乘区拆解：逐项展示每个倍率来源与取值（便于校验）
5. 调试信息（可折叠）：所用 buff ID 与来源表

## 7.3 交互规则

1. 切换敌人、天色、阶段时，一次性重算主线/再战/死斗全部输出矩阵。
2. 非法组合显示 `N/A` 并给出原因（例如敌人无再战/死斗数据）。
3. 数据加载失败时显示可读错误，不静默失败。

## 10. 验收标准

1. 同一输入条件下，前端结果与 Excel 样例偏差为 0（取整后一致）。
2. 主线/再战/死斗三类结果矩阵均可正确生成并一致可复现。
3. 特殊敌人（狮子猿、怨鬼、心中弦一郎）可按特例 buff 正确出值。
4. 数据源不可达时页面有明确错误反馈。
5. 计算链路可追溯（可看到每个倍率来源）。

## 12. 完整实现提示词（供后续开发模型直接使用）

你是资深前端工程师与数据建模工程师。  
请为“只狼 BWiki 数据计算器”实现一个可部署到 MediaWiki 页面中的前端计算器。  
注意：本任务目标是“计算器”，不包含基础数据展板与伤害计算器。

### 已知背景

1. 运行环境：MediaWiki 页面（偏静态前端环境）。
2. 数据来源：外部静态 JSON（由 Excel 清洗后发布在 GitHub Pages 或其他静态托管）。
3. 核心需求：根据敌人基础值与多个 buff 乘区，计算 HP、躯干、躯干恢复。

### 你必须遵守的实现要求

1. 使用纯前端方案（HTML + CSS + JavaScript）。
2. 计算逻辑独立封装成纯函数，UI 与计算解耦。
3. 提供完整错误处理（网络失败、字段缺失、非法输入）。
4. 输出不仅有最终值，还要有“倍率分解明细”。
5. 保证切换参数时即时重算，结果稳定可复现。
6. 与给定公式严格一致，最终结果向下取整。
7. 不得把数据库硬编码在页面中，必须通过 URL 拉取。

### 计算规则（必须严格实现）

1. 一周目主线
- 普通：`Base * A_time`
- 钟鬼：`Base * A_time(time_seq+1)`
- 交符：`Base * A_time_1ng_charm * A_enemy_type`
- 双难：`Base * A_time_1ng_charm(time_seq+1) * A_enemy_type`

2. 多周目主线（2~8）
- 普通：`Base * A_time * A_multi * A_ng(normal_index)`
- 钟鬼：`Base * A_time(time_seq+1) * A_multi(time_seq+1) * A_ng(normal_index)`
- 交符：`Base * A_time * A_multi * A_ng(charm_index) * A_enemy_type`
- 双难：`Base * A_time(time_seq+1) * A_multi(time_seq+1) * A_ng(charm_index) * A_enemy_type`

3. 再战
- 普通：`Base * A_mind[0] * A_re[0]`
- 钟鬼：`Base * A_mind[3] * A_re[3]`
- 交符：`Base * A_mind[2] * A_re[2]`
- 双难：`Base * A_mind[1] * A_re[1]`

4. 死斗
- 普通：`Base * A_mind[0] * A_death[0]`
- 钟鬼：`Base * A_mind[3] * A_death[3]`
- 交符：`Base * A_mind[2] * A_death[2]`
- 双难：`Base * A_mind[1] * A_death[1]`

5. 特殊叠乘
- 当 `selected_phase >= buff_active_phase_num` 且存在 `phase_change_buff` 时，额外乘阶段倍率。
- `special_buff` 的 `hp_bonus` 与 `stamina_bonus` 分开作用到 HP 与躯干相关值。
- `enemy_type_buff` 仅在交符/双难应用。
- 再战/死斗数据列仅对enemy_type_buff为7903的敌人生效，其余敌人不显示这两列数据，比较特别的，怨鬼由于连战数据也不一样，额外加一列修罗连战数据

6. 输出取整
- 最终显示值全部 `Math.floor`（向下取整）。
