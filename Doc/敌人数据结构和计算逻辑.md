# 敌人数据结构

1. name

   1. 敌人名称
2. select_menu和npc_param的id

   1. 唯一标识ID，所有的数据查询以这个为入口
3. 基础HP上限（int）

   1. base_hp
4. 基础躯干上限（int）

   1. base_stamina
5. 基础躯干恢复（int）

   1. base_stamina_recover
6. 区域天色加成(list)

   1. base_bonus_buff
7. 区域天色加成__一周目交符（list）

   1. base_bonus_buff_NG1_no_charm
8. 多周目平衡（list）

   1. multi_NG_balance_buff
9. 转阶段特效（int）

   * phase_change_buff
   * 一些敌人转阶段会带上的特效
   * key是对应的阶段，value是对应的加成ID
10. 特殊加成（int）

    * special_buff
    * 一些敌人的特殊加成，所有阶段都存在，比如水生氏成的红眼加成
12. 心中战场/再战/死斗特效（list）

    1. mind_battlefield_buff
    2. re_battle_buff
    3. death_battle_buff
    4. 对于怨鬼，连战的buff和再战不一样所以单独填两个
13. 兵种特效（int）

    1. enemy_type_buff
    2. 只在交符后生效，即只对交符和双难两个难度生效
14. 天色序列（list）

    1. time_seq
    2. 四个天色分别0123，对应清晨，正午，黄昏，深夜
    3. 天色序列下拉菜单能切换哪几个天色
15. 周目成长特效（list）
    1. NG_buff
    2. 7+7，交符序列直接+7
    3. 因为1周目没有加成，所以一组数据只有7，加上交符的7个整个的list长度就是14，2周目的要对应list的第一个数据应该是list[0]，所以让NG_num作为index调用数据的话就是NG_buff[NG_num-2]
16. 阶段数

    1. phase_num
    2. 确定阶段的下拉菜单用，数字多少就显示多少个阶段选项，例如2，就显示一阶段，二阶段，3就显示为一阶段，二阶段，三阶段

## 调用逻辑

### 周目加成list确定

加成是一个序列，就是列表list，但是只会填写序列开始的第一个加成的ID，要根据这个ID对应的下一行ID的不同，来确定整个list，分以下情况（以base_bonus_buff为例）

1. 下一行ID结尾不是1也不是4，这时候加成序列的length只有1，即只包含填写的这个加成，例如7000
2. 结尾是4，这时候加成序列的length是2，包含填写的ID和这个ID，例如7020和7024
3. 最后一种情况就是下一个ID结尾是1，这种情况加成序列length为5，比如7010,7011,7012,7013,7014
4. 然后其余两个全部以base_bonus_buff做锚定去调用，base_bonus_buff_NG1_no_charm=base_bonus_buff+700，multi_NG_balance_buff=base_bonus_buff+500

### 再战、连战、死斗加成list确定

就是mind_battlefield_buff、re_battle_buff、death_battle_buff、shura_battlefield_buff、shura_battle_buff这五个

这五个统一list长度为4，即填写的ID加上后面跟着的三个，例如7950、7951、7952、7953

### 表格的数据逻辑

1. 周目加成的调用逻辑：选择天色后，调用对应time_seq结尾的ID，例如清晨，则调用0的结尾ID，例如7010，黄昏则调用2的结尾ID，例如7012，这适用`普通`和`交符`难度，对于钟鬼和双难，在普通和交符的加成ID上+1，如果序列中找不到对应的ID，就和普通和交符使用一样的加成ID
2. phase_change_buff每个阶段都是独立的，只对对应阶段生效，不会保留到其他阶段，列表中的key代表生效阶段，value代表调用的加成ID
3. special_buff对所有难度生效，不过special_buff的stamina_bonus增益可能不和hp_bonus一样，所以要分别进行乘算，这个没在下面计算规则的表格里写出来，但有这个乘区存在，如果有的话就生效
4. 敲钟和钟鬼是同义词

### 数据调用逻辑链

1. 一切从select_menu的ID出发，根据ID去npcparam的对应ID找各种基础数据，包括phase_num，base_hp，base_stamina，base_stamina_recover，base_bonus_buff，multi_NG_balance_buff，enemy_type_buff，buff类的拿到ID之后去同名分表里读取对应数据
2. 比较特殊的，phase_change_buff，special_buff我写在了select_menu里
3. 除了NG_buff在同名分表里调用，其余的加成全部去speffect_param里根据ID读取对应数据

## 计算规则

### 一周目

| 难度 | 公式                                                         |
| ---- | ------------------------------------------------------------ |
| 普通 | base_[hp/stamina/stamina_recover]\*base_bonus_buff[time_seq] |
| 钟鬼 | base_[hp/stamina/stamina_recover]\*base_bonus_buff[time_seq+1] |
| 交符 | base_[hp/stamina/stamina_recover]\*base_bonus_buff_NG1_no_charm[time_seq]\*enemy_type_buff |
| 双难 | base_[hp/stamina/stamina_recover]\*base_bonus_buff_NG1_no_charm[time_seq+1]\*enemy_type_buff |

### 多周目（2-8）

| 难度 | 公式                                                         |
| ---- | ------------------------------------------------------------ |
| 普通 | base_[hp/stamina/stamina_recover]\*base_bonus_buff[time_seq]*multi_NG_balance_buff[time_seq]\*NG_buff[ng_num-2] |
| 钟鬼 | base_[hp/stamina/stamina_recover]\*base_bonus_buff[time_seq+1]\*multi_NG_balance_buff[time_seq+1]\*NG_buff[ng_num-2] |
| 交符 | base_[hp/stamina/stamina_recover]\*base_bonus_buff[time_seq]*multi_NG_balance_buff[time_seq]\*NG_buff[ng_num-2+7]\*enemy_type_buff |
| 双难 | base_[hp/stamina/stamina_recover]\*base_bonus_buff[time_seq+1]\*multi_NG_balance_buff[time_seq+1]\*NG_buff[ng_num-2+7]\*enemy_type_buff |

这里的ng_num指周目数

### 再战

| 难度 | 公式                                                         |
| ---- | ------------------------------------------------------------ |
| 普通 | base_[hp/stamina/stamina_recover]\*mind_battlefield_buff[0]\*re_battle_buff[0] |
| 钟鬼 | base_[hp/stamina/stamina_recover]\*mind_battlefield_buff[3]\*re_battle_buff[3] |
| 交符 | base_[hp/stamina/stamina_recover]\*mind_battlefield_buff[2]\*re_battle_buff[2] |
| 双难 | base_[hp/stamina/stamina_recover]\*mind_battlefield_buff[1]\*re_battle_buff[1] |

### 死斗

| 难度 | 公式                                                         |
| ---- | ------------------------------------------------------------ |
| 普通 | base_[hp/stamina/stamina_recover]\*mind_battlefield_buff[0]\*death_battle_buff[0] |
| 钟鬼 | base_[hp/stamina/stamina_recover]\*mind_battlefield_buff[3]\*death_battle_buff[3] |
| 交符 | base_[hp/stamina/stamina_recover]\*mind_battlefield_buff[2]\*death_battle_buff[2] |
| 双难 | base_[hp/stamina/stamina_recover]\*mind_battlefield_buff[1]\*death_battle_buff[1] |

```
注：因为再战和死斗buffID的排列顺序是普通，双难，交符，钟鬼，跟列表展示顺序有差异，所以改变再战和死斗的buffID调用顺序来适应表格
```

### 怨鬼修罗连战计算

| 难度 | 公式                                                         |
| ---- | ------------------------------------------------------------ |
| 普通 | base_[hp/stamina/stamina_recover]\*shura_battlefield_buff[0]\*shura_battle_buff[0] |
| 钟鬼 | base_[hp/stamina/stamina_recover]\*shura_battlefield_buff[3]\*shura_battle_buff[3] |
| 交符 | base_[hp/stamina/stamina_recover]\*shura_battlefield_buff[2]*shura_battle_buff[2] |
| 双难 | base_[hp/stamina/stamina_recover]\*shura_battlefield_buff[1]\*shura_battle_buff[1] |

### base_bonus_buff的list length只有2的特殊情况

在黄昏天色，就是天色序号2的时候，普通数据=钟鬼数据，交符数据=双难数据

即条件满足==base_bonus_buff的list长度= 2 and time_seq <= 2==的时候，表格的数据计算逻辑如下

#### 1周目

| 难度      | 公式                                                         |
| --------- | ------------------------------------------------------------ |
| 普通/钟鬼 | base_[hp/stamina/stamina_recover]*base_bonus_buff[time_seq]  |
| 交符/双难 | base_[hp/stamina/stamina_recover]\*base_bonus_buff_NG1_no_charm[time_seq]\*enemy_type_buff |

#### 2-8周目

| 难度      | 公式                                                         |
| --------- | ------------------------------------------------------------ |
| 普通/钟鬼 | base_[hp/stamina/stamina_recover]\*base_bonus_buff[time_seq]\*multi_NG_balance_buff[time_seq]*NG_buff[ng_num-1] |
| 交符/双难 | base_[hp/stamina/stamina_recover]\*base_bonus_buff[time_seq]\*multi_NG_balance_buff[time_seq]\*NG_buff[ng_num-1+7]*enemy_type_buff |
